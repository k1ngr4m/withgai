# 06_GAME_FLOW_AND_BACKEND_GUIDE

## 文档定位

本文用于说明《withgai》当前 First Playable 版本的完整游戏流程，并详细整理当前唯一公开可玩的职业：`后端`。

本文信息来源于当前工程中的设计文档、数据表与运行时实现：

- `Resources/Docs/01_GAME_DESIGN_BRIEF.md`
- `Resources/Docs/02_SYSTEM_SPEC.md`
- `DataTables/Datas/*.csv`
- `Data/Generated/Config/game_config.json`
- `Scripts/Services/*.gd`

当前实现范围是：`后端`职业完整可玩；`前端 / 测试 / 算法 / 产品经理 / HR` 在 UI 中可见，但作为锁定占位，不进入公开单局的奖励、商店和战斗池。

## 1. 游戏总览

《withgai》是一款以现代写字楼为舞台的单人肉鸽卡牌游戏。玩家选择一个职场职业，从基层办公区一路向上爬楼，通过路线选择、回合制卡牌战斗、奖励、商店、事件、休息和局外成长，最终挑战顶层制度化 Boss。

核心体验可以概括为：

- 选择职业，拿到初始牌组和初始遗物。
- 在树状楼层地图中选择路线。
- 进入战斗，读敌方意图，用卡牌构筑职业引擎。
- 战斗后获得卡牌、绩效点和可能的遗物。
- 通过商店、事件和休息处调整牌组。
- 击败每章 Boss 后进入下一章。
- 胜利或失败后结算窝囊费，用于局外工位升级。

## 3. 关键资源术语

| 通用名 | 游戏内包装 | 说明 |
| --- | --- | --- |
| HP | 精神状态 | 单局生命值。归零则本局失败。 |
| Energy | 精力 | 每回合打牌资源，默认基础值为 3。 |
| Block | 防线 | 抵挡本回合敌方攻击，通常回合开始清空。 |

## 4. 单局结构

单局采用 3 章爬楼结构。当前目标单局时长设计为 40-60 分钟。

| 章节 | 楼层范围 | 区域名 | 主题 | Boss |
| --- | --- | --- | --- | --- |
| 第一章 | 1F-6F | 基层办公区 | 新人杂活、低级流程、日常压榨 | 画饼主管 |
| 第二章 | 7F-12F | 中层管理区 | 汇报体系、绩效压迫、审批迷宫 | 变异HR |
| 第三章 | 13F-18F | 总裁区 | 企业意志、制度实体化、终极会议 | 变异总裁 |

每章地图由 `MapService` 生成，节点类型来自 `MapNodeDef.csv`。

| 节点类型 | 权重 | 作用 |
| --- | ---: | --- |
| 普通战斗 | 45 | 主要构筑来源，胜利后进入奖励页。 |
| 精英战斗 | 12 | 更高风险，胜利后有遗物候选，并可触发精英计数。 |
| 休息处 | 14 | 恢复精神状态或升级一张牌。 |
| 商店 | 12 | 购买卡牌、遗物、移除卡牌、刷新库存。 |
| 随机事件 | 17 | 在收益、代价和牌组调整之间做选择。 |
| Boss | 1 | 章节终点，击败后进入下一章或结算胜利。 |

## 5. 从启动到结算的完整流程

### 5.1 主菜单

进入游戏后首先来到主菜单。主菜单负责呈现当前 First Playable 状态：

- 后端是当前唯一可公开开局的职业。
- 其他职业以锁定占位形式展示。
- 可直接快速开始后端，也可以进入职业选择页。
- 可以继续中断存档。
- 可以进入局外成长页。
- 可以打开设置面板，调整全屏、主音量、减少动效、环境动效和屏幕震动等设置。

### 5.2 职业选择

职业选择页展示所有职业的职业定位、锁定状态、解锁说明和职业树进度。

当前公开开局只允许选择 `后端`。如果尝试从服务层创建锁定职业单局，`RunSession.create_new_run()` 会拒绝，并保留已有后端运行状态。

后端开局时获得：

- 初始精神状态：72
- 基础精力：3
- 初始遗物：灰度发布开关
- 初始牌组：10 张后端牌
- 起始章节：第 1 章
- 起始楼层：1F
- 初始场景标识：`map`

局外升级会在开局时立刻生效，例如人体工学椅提高最大精神状态，极品咖啡豆在满级时提高基础精力，扩容硬盘增加首回合抽牌。

### 5.3 地图路线选择

地图页展示当前章节、楼层节点和可达路线。

当前地图交互是“先预览，再确认进入”：

1. 玩家点击可达节点。
2. 右侧或详情面板显示楼层、节点类型、预期奖励和后续路线数量。
3. 玩家点击确认按钮进入该节点。

节点完成后，`MapService` 会标记访问状态，刷新可达节点，并在 Boss 节点完成时推进章节。

### 5.4 战斗节点

普通战斗、精英战斗和 Boss 都进入战斗页。

战斗页核心区域包括：

- 玩家区域
- 职业资源面板
- 敌人区域
- 敌方意图区域
- 手牌区域
- 战斗日志
- 结束回合按钮

每场战斗开始时，系统会：

- 根据当前牌组生成抽牌堆。
- 应用开局遗物和局外升级。
- 设置玩家当前精力。
- 抽起始手牌，默认 5 张，受遗物和局外升级影响。
- 生成敌人意图。

### 5.5 战斗回合循环

一个标准回合按以下顺序推进：

1. 玩家回合开始：
   - 非首回合时清空防线。
   - 重置每回合一次的遗物和状态触发标记。
   - 设置当前精力为基础精力。
   - 结算回合开始状态，例如焦虑、加班、Redis 预热、API 网关、服务在线。
   - 刷新敌方意图。
   - 抽牌。
2. 玩家阶段：
   - 选择目标。
   - 打出攻击、技能或能力牌。
   - 消耗精力。
   - 触发卡牌效果、状态、遗物和职业资源变化。
   - 卡牌进入弃牌堆。
3. 玩家结束回合：
   - 结算回合末状态，例如消息队列、服务巡检。
   - 弃掉未保留手牌。
4. 敌人阶段：
   - 敌人按可见意图行动。
   - 可能攻击、格挡、施加负面、污染牌组、召唤、清除玩家增益或触发阶段变化。
5. 胜负检查：
   - 所有敌人被击败则进入奖励。
   - 玩家精神状态归零则进入结算失败。

### 5.6 战斗奖励

战斗胜利后会生成 `pending_reward_state`，进入奖励页。

默认奖励规则：

- 候选卡牌：从当前职业池与共享池中抽取 3 张。
- 绩效点：18-35。
- 精英战斗额外绩效点：+20。
- 精英/Boss 可能提供遗物候选，最多 2 个。
- 如果拥有园区停车月卡，精英奖励额外 +15 绩效点。

奖励页允许：

- 选择一张候选卡加入牌组。
- 跳过卡牌。
- 选择一个候选遗物。
- 跳过遗物。
- 确认后回到地图、进入下一章或进入结算。

### 5.7 商店节点

商店页提供三类核心操作：

- 买卡牌。
- 买遗物。
- 移除一张牌。
- 刷新库存。

当前费用：

| 项目 | 基础费用 |
| --- | ---: |
| 购买卡牌 | 45 绩效点 |
| 购买遗物 | 85 绩效点 |
| 移除卡牌 | 75 绩效点 |
| 刷新库存 | 20 绩效点 |

商店库存从当前职业池、共享池和可用遗物池中生成。已拥有的遗物不会重复进入购买结果。

折扣来源：

- 员工内购券：首次商店消费折扣。
- 员工食堂月卡：局外升级，每级降低首次消费成本。

### 5.8 随机事件

事件页提供一段事件文本和多个选项。选项会直接修改单局状态。

当前事件包括：

| 事件 | 主要选择方向 |
| --- | --- |
| 老板办公室门没锁 | 获得绩效点，或升级一张牌。 |
| 内网邮件误抄送 | 抽牌式收益，或获得绩效点。 |
| 茶水间八卦 | 恢复精神状态，或移除废牌。 |
| 深夜自动贩卖机故障 | 花绩效点换随机卡，或直接拿绩效点。 |
| 帮实习生背锅 | 损失精神状态换随机卡，或获得绩效点。 |
| 年会抽奖预演 | 高额绩效点，或小幅恢复。 |
| 领导单独谈话 | 获得绩效点，或获得随机遗物。 |
| 裁员传闻 | 恢复精神状态，或获得绩效点。 |

事件结算后会记录事件历史，并回到地图。

### 5.9 休息处

休息处提供两个选择：

- 冥想：恢复精神状态。
- 复盘：升级一张牌。

恢复量为：

```text
最大精神状态的 30% + 午休折叠床等级 * 4
```

升级牌的运行时规则：

- 非 X 费牌的基础费用降低 1，最低为 0。
- 对以下效果类型，如果参数中有 `amount`，则 `amount +2`：
  - `deal_damage`
  - `gain_block`
  - `add_cache`
  - `add_component`
  - `add_style_layer`
  - `add_compute`
  - `add_case`
  - `inject_bug`
- 部分复合型专用效果，例如 `circuit_breaker`、`service_degrade`，当前主要吃到费用降低；其内部参数不是通过通用升级列表逐项增加。

### 5.10 中断续玩

战斗、奖励、商店、事件和休息页面都支持保存并返回主菜单。

当前中断存档会保存：

- 当前场景标识。
- 单局状态。
- 战斗中的敌人、手牌、牌堆、精力、防线、目标选择和战斗日志。
- 奖励候选。
- 商店库存、已购买内容和刷新次数。
- 事件准备状态。
- 休息处当前精神状态。

恢复时，顶层 `scene_tag` 是权威恢复目的地；未知或旧版本场景标识会回退到地图，避免卡在主菜单。

### 5.11 结算与局外成长

当玩家失败或击败第三章 Boss 后进入结算页。

结算会统计：

- 最高楼层。
- 击败 Boss 数。
- 战斗胜利数。
- 精英胜利数。
- 事件、商店、休息次数。
- 击败敌人数。
- 是否胜利。
- 获得窝囊费。

窝囊费计算：

```text
到达楼层 * 2 + 击败 Boss 数 * 35 + 胜利奖励 80
```

局外成长包括：

| 升级 | 最高等级 | 费用曲线 | 效果 |
| --- | ---: | --- | --- |
| 人体工学椅 | 3 | 20 / 40 / 80 | 开局最大精神状态每级 +4。 |
| 防窥膜 | 3 | 20 / 45 / 90 | 战斗开始每级 +2 防线。 |
| 极品咖啡豆 | 2 | 30 / 60 | 满级时基础精力 +1。 |
| 扩容硬盘 | 2 | 30 / 60 | 首回合每级额外抽 1 张。 |
| 午休折叠床 | 3 | 20 / 45 / 90 | 休息处恢复量每级 +4。 |
| 员工食堂月卡 | 2 | 25 / 50 | 商店首次消费每级折扣 5。 |

职业解锁树当前以展示为主。后端默认开放；其他职业虽有长期解锁条件文案和里程碑显示，但 First Playable 入口仍只允许后端公开开局。

### 7.2 职业幻想

后端不是最快启动的职业，而是逐步把生产环境“搭起来”的职业。前几回合通过部署服务、积累缓存和稳定防线争取时间；中盘靠服务在线、API 网关、分库分表等持续收益滚雪球；后期用全量回写等牌把缓存转换成爆发伤害。

这个职业的体验关键词是：

- 稳健
- 防守
- 累积
- 服务引擎
- 中后期压制
- 缓存爆发

### 7.3 后端核心资源

#### 服务

服务代表已经上线的后端系统。服务可以来自：

- `发布脚本` 的 `deploy_service`。
- `容器编排 / 集群扩容 / 零停机升级` 施加的 `service_online` 状态。

运行时会从职业资源 `services` 和状态 `service_online` 中取较高值，避免重复计算。

服务在线的回合效果：

- 玩家回合开始：每层服务获得 1 缓存和 2 防线。
- 玩家回合结束：每层服务对所有存活敌人造成 2 伤害。

服务越多，后端越接近“什么都不做也能产生收益”的引擎状态。

#### 缓存

缓存是后端的可累积职业资源。大量后端卡牌会在攻击或防守的同时获得缓存。

缓存的主要用途：

- 提供资源面板上的长期积累感。
- 被 `全量回写` 消耗，转换为高额单体伤害。
- 触发 `分库分表` 的额外缓存收益。

当前缓存状态也声明了 `deal_damage` hook，用于支持缓存相关伤害结算。

#### 请求

`消息队列堆积` 会施加 `request_queue`，显示为请求。

请求的回合末效果：

- 每层请求对所有存活敌人造成 3 点伤害。
- 结算后消耗请求层数。

请求适合对多敌人战斗施压，也能在玩家结束回合后补刀。

### 7.4 后端相关状态

| 状态 | 显示 | 触发 | 参数 | 说明 |
| --- | --- | --- | --- | --- |
| `service_online` | 显示 | 回合开始、回合结束 | 无 | 服务在线。回合开始给缓存和防线，回合结束造成群体伤害。 |
| `cache` | 显示 | 造成伤害 | 无 | 缓存资源，用于后端爆发。 |
| `api_gateway` | 显示 | 回合开始 | 防线 4，服务阈值 2，抽牌 1 | 每回合开始给防线；服务不少于 2 时额外抽牌。 |
| `redis_warmup` | 隐藏 | 回合开始 | 减费 1 | 本回合打出后，下回合转化为减费。 |
| `cost_reduction` | 隐藏 | 卡牌费用 | 无 | 降低下一张非 X 费牌费用，使用后消耗。 |
| `request_queue` | 显示 | 回合结束 | 每层 3 伤害 | 回合末按请求数打全体，结算后消耗。 |
| `sharding` | 显示 | 获得缓存 | 额外缓存 1 | 每回合第一次获得缓存时额外获得缓存。 |

## 11. 后端构筑方向

### 11.1 服务引擎

核心牌：

- 发布脚本
- 容器编排
- 集群扩容
- 零停机升级
- API网关
- 追踪链路

思路：

1. 尽早部署服务。
2. 用追踪链路找到服务相关牌。
3. 服务在线后，回合开始自动获得缓存和防线。
4. 回合结束服务巡检造成全体伤害。
5. API 网关在服务达到阈值后提供额外抽牌。

优势是越拖越稳，适合打 Boss 和长战斗。弱点是前期如果服务牌不上手，启动会偏慢。

### 11.2 缓存爆发

核心牌：

- 接口打点
- 慢查询优化
- Redis预热
- 分库分表
- 全量回写

思路：

1. 用低费攻击和防守牌不断积累缓存。
2. 分库分表让每回合第一次缓存收益变高。
3. Redis 预热提供大量缓存并改善下回合费用。
4. 最后用全量回写消耗缓存打爆发。

全量回写的基础伤害是 14，每层缓存追加 3 伤害。缓存越多，终结越干脆。

### 11.3 高压防守

核心牌：

- 熔断保护
- 服务降级
- 流量削峰
- 双机热备
- 无损发布
- 灾备接管
- 升降桌

思路：

1. 读敌方意图，优先处理高攻击回合。
2. 熔断保护在敌方总攻击达到阈值时额外给防线。
3. 服务降级直接降低所有攻击/多段攻击意图。
4. 流量削峰把已承受压力转成缓存。
5. 高防线牌配合升降桌获得额外防线。

这个方向适合稳过精英和 Boss，但需要补足伤害终结手段。

### 11.4 队列群伤

核心牌：

- 消息队列堆积
- 服务在线类能力牌
- 请求聚合
- 全量回写

思路：

1. 用消息队列堆积制造请求。
2. 请求在回合末对所有敌人造成伤害。
3. 服务在线本身也在回合末造成全体服务巡检伤害。
4. 对多敌人战斗尤其有效。

这套不一定立刻爆发，但能在结束回合后持续清理敌人。

## 12. 后端对局节奏建议

### 12.1 前期

优先目标：

- 保持精神状态健康。
- 尽量打出发布脚本。
- 不要过早拿太多高费能力牌。
- 选择能稳定给防线、缓存或抽牌的牌。

推荐早期拿牌：

- 缓存命中
- 热修复回滚
- 服务降级
- 慢查询优化
- 追踪链路

### 12.2 中期

优先目标：

- 建立至少 1-2 层服务。
- 找到一个长期能力：API网关、分库分表、容器编排、集群扩容。
- 开始考虑全量回写作为终结牌。
- 在休息处升级核心低费牌或核心输出牌。

### 12.3 后期

优先目标：

- 服务和缓存形成双引擎。
- 用 API 网关补抽牌。
- 用服务降级和高防线牌处理 Boss 大攻击。
- 用全量回写或服务巡检完成终结。

后期后端的理想状态是：回合开始自动获得资源，回合内用牌保护自己和继续堆缓存，回合结束还能自动造成伤害。

## 13. 后端弱点与风险

| 风险 | 表现 | 应对 |
| --- | --- | --- |
| 启动慢 | 服务牌不上手，前几回合只能靠基础攻防。 | 追踪链路、蓝光眼镜、热修复回滚、晨会同步。 |
| 爆发依赖缓存 | 没有缓存时全量回写只是普通攻击。 | 多拿缓存生成牌，使用分库分表和 Redis预热。 |
| 高费能力卡卡手 | 能力牌过多时短期防守不足。 | 保持低费防守比例，不要过早贪多张 2-3 费能力。 |
| 污染牌干扰循环 | Boss 和部分敌人会塞污染或诅咒。 | 回滚版本、商店移除、事件移除、休息升级优先处理核心牌。 |
| 多段攻击压力 | 多段攻击会持续吃防线。 | 服务降级、熔断保护、升降桌、高防线牌。 |

## 14. 后端推荐升级优先级

推荐优先升级：

- 接口打点：费用从 1 降到 0，伤害和缓存都提升，早期收益很高。
- 发布脚本：费用从 1 降到 0，部署服务更容易。
- 热修复回滚：本来 0 费，升级主要提高防线和缓存，是节奏牌。
- 慢查询优化：伤害和缓存提升，适合缓存爆发路线。
- 全量回写：费用降低且基础伤害提升，是后期终结牌。

可以按构筑情况升级：

- 熔断保护：升级后费用降低，但当前复合效果参数不全走通用 `amount +2`，主要价值是更容易打出。
- 服务降级：费用已经是 0，升级收益有限，除非需要更高基础生存稳定性。
- 高费防守牌：无损发布、灾备接管、双机热备升级后费用更顺。

## 15. 后端一句话教学稿

```text
后端通过部署服务和积累缓存取得长期优势。服务会在回合开始提供缓存和防线，并在回合结束自动伤害敌人；缓存可以被全量回写消耗，转化为高额爆发。
```

更详细的教程节奏可以写成：

```text
先用发布脚本部署服务，再用防守牌稳住局面。服务越多，每回合自动获得的缓存和防线越多。缓存足够后，用全量回写完成终结。API网关、分库分表和Redis预热能让你的系统越跑越快。
```
